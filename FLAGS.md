# FLAGS

When provided with this flags be sure to do accordingly.


## **--adversary**

- Act as an adversary agent
- Question everything to try to get better output and performance with lower complexity

## **--simple**

- Direct fix with validation
- Small modifications
- Short answers

## **--medium**

- Plan-Build-Review cycle
- Medium modifications
- Medium answers

## **--complex**

- Full feature workflow
- Large modifications
- Big answers

The following flags are mutually exclusive (`--plan`, `--build`):

## **--plan**

- Just plan the task dont implement.
- Carefully think and elaborate a plan to provide for the main agent.
- Follow yaml-structured output style.

## **--build**

- You should carefully try to implement the task at hand.
- Follow yaml-structured output style.
